# Product Backlog - Tutorio

## Story 1
**User Story:**  
As a student, I want to register an account so that I can access tutoring services.

**Priority:** High  
**Story Points:** 3

**Acceptance Criteria:**
- User can input name, email, password
- Email validation is enforced
- Password must meet security requirements
- Account is saved in database

---

## Story 2
**User Story:**  
As a tutor, I want to register separately so that I can offer tutoring sessions.

**Priority:** High  
**Story Points:** 5

**Acceptance Criteria:**
- Tutor registration form is separate
- Tutor can add subject expertise
- Tutor profile is stored
- Validation prevents duplicate email

---

## Story 3
**User Story:**  
As a user, I want to log in securely so that I can access my dashboard.

**Priority:** High  
**Story Points:** 5

**Acceptance Criteria:**
- JWT authentication works
- Invalid credentials show error
- Successful login redirects to dashboard

---

## Story 4
